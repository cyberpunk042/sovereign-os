Do NOT minimize anything. Deliver the FULL, comprehensive scope of what the operator asks — never a reduced, sampled, partial, or "good enough" subset. When in doubt, do it completely and do more, not less.

**Why:** New rule from the operator on 2026-07-03, and the project's own sacrosanct standing rule ("We do not minimize anything" — the `x-sovereign-standing-rule` meta tag). The operator has repeatedly caught me delivering reduced-scope work: short blurbs instead of real spec purposes, partial feature lists, incremental patches instead of the full plan. Minimizing is a core failure mode.

**How to apply:** Cover every item, every angle, the whole surface. If the ask implies N things, do all N. Match the operator's "1000 hours / a lot on the plate" ambition — comprehensive, not token. If full scope is too large for one pass, do it in full-scope slices and say what remains — never silently shrink. See [[operator-is-always-the-driver]].
