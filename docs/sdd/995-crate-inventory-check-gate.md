# SDD-995 — the crate-inventory generator gains a `--check` freshness gate + sync lint (F-2026-098)

> Status: draft
> Owner: operator-directed 2026-07-14 (phase-1 audit continuation, "continue"); agent-authored.
> Closes: **F-2026-098** (LOW) — the one generated-doc script without a `--check` gate.
> Mandate module: **E11.M995**.
> Number band: **950–999 (phase-1 audit session)** per SDD-100.

## The gap

The repo generates several living docs from the tree and guards each against drift
with a regen-and-compare gate: `gen-sdd-catalog.py --check` (the mdbook catalog,
SDD-958), the counts-contract (SDD-952), the island register (SDD-955), the
per-milestone roll-up (SDD-972), the runtime/verify/phases render scripts. Every one
exposes a `render()` and a `--check` mode, and a lint runs `--check` in CI so the
page can only be regenerated, never hand-edited or left stale.

`scripts/docs/gen-crate-inventory.py` — which generates
`docs/architecture/crate-inventory.md`, the complete map of every workspace crate
and whether anything reaches it — was the **lone exception**. Its `main()` built the
page inline and **unconditionally wrote** it; there was no `--check` flag and **no
lint** guarding the committed page. A crate added, removed, or re-described in its
`Cargo.toml` would silently leave the inventory stale until someone remembered to
rerun the script — exactly the living-doc drift the audit (F-2026-033 family) warns
about, on the one generator that had slipped the net.

## The fix — render/check split + a sync lint (the house pattern)

1. **`gen-crate-inventory.py`**: the page-building body of `main()` is factored into
   `render() -> str` (pure — computes the crate buckets and returns the full Markdown).
   `main()` now:
   ```python
   check = "--check" in sys.argv[1:]
   rendered = render()
   current = OUT.read_text(...) if OUT.exists() else None
   if rendered == current:
       return 0                       # up to date
   if check:
       print("DRIFT: … regenerate with …"); return 1
   OUT.write_text(rendered, ...)      # default: (re)write
   return 0
   ```
   Same regen-and-compare shape as `gen-sdd-catalog.py`.

2. **`tests/lint/test_crate_inventory_sync.py`** (4 cases): the generator exists +
   exposes `render()` + supports `--check`; the committed page **equals** `render()`
   (drift → CI fail with a regenerate hint); and the page carries its
   `GENERATED by …` do-not-hand-edit banner.

No behavioural change to what the inventory *contains* — the same `render()` output
as before; this only makes staleness a CI failure instead of a silent gap.

## Verification (real, observed)

- `python3 scripts/docs/gen-crate-inventory.py` → `already up to date` (the committed
  page was current); `--check` on the synced tree → **exit 0**; `--check` after
  appending a stray line → prints `DRIFT: …` and **exit 1** (both directions proven);
  page restored.
- `python3 -m pytest tests/lint/test_crate_inventory_sync.py` → **4 passed**.
- `ruff check scripts/docs/gen-crate-inventory.py tests/lint/test_crate_inventory_sync.py`
  → clean.
- Full `tests/lint` green.

## Scope / safety

`scripts/docs/gen-crate-inventory.py` (render/main split + `--check`) + a new
`tests/lint/` file + this SDD + registries. No change to the generated inventory's
content, no crate, no runtime, no new dependency. Collision-safe. R10212 / MS043
untouched. MS003 `unsigned-pending-MS003`.

## Non-goals

- Wiring the generator into a `make docs` / pre-push hook (the CI lint is the gate;
  a hook is a separate convenience).
- The broader "413 unwired cockpit crates" question the inventory *documents*
  (F-2026-001 — the wasm bridge is chipping at it); this only keeps the map honest.
