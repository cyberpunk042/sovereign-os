# sovereign-cockpit-* consumption map

> **Generated by `scripts/cockpit/gen-consumption-map.py`** (CI-locked by `tests/lint/test_cockpit_consumption_map.py`). Makes visible which of the cockpit micro-crates are actually consumed vs compiled-and-tested-only. **Non-destructive** — the "we do not minimize" doctrine stands; this is the decision surface for any operator-ratified wire/retire, not an action. Regenerate: `python3 scripts/cockpit/gen-consumption-map.py`.

**418 cockpit crates** · **55 live-consumed** (a real product path calls in) · **363 export-only** (a `validate` bridge is exported but the committed webapp invokes ~none — the full bridge `cockpit_wasm_full.js` is not committed) · **0 compiled-only** (nothing references them at all).

> The July-2026 audit's "~380 shelfware" figure ≈ the export-only + compiled-only tiers: crates CI builds and tests but that no shipped panel actually drives. They are not dead (validators run in tests) and not deleted ("we do not minimize") — they are the wire-or-retire backlog this map makes queryable.
