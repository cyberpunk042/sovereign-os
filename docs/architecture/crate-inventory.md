#### `token-·` (4)

- **`sovereign-token-grammar-mask`** — Bridges a character-level context-free grammar to token-level decoding: given the text generated so far, computes the boolean mask of which vocabulary tokens keep a valid parse alive, plus whether end-of-sequence is allowed. The piece that makes grammar-constrained generation work on a real tokenizer — a model emits tokens, not characters — by feeding each candidate token's string through the Earley recognizer with a cheap first-character prefilter. - **`sovereign-token-law-deny`** — The NEGATIVE token-law plane: forbid a running model from ever emitting text that contains any of a set of denied substrings (safety/injection/toxicity phrases). A substring ban is not a per-token property — a forbidden phrase can span token boundaries — so this drives an Aho-Corasick automaton incrementally from the committed generation and bans exactly the tokens that would COMPLETE a banned match. safe_token_ids returns the allow-list (all tokens minus the completing ones), the same shape sovereign-token-law-mask's TokenLawPlanes consumes, so it composes with grammar/regex/policy planes. Composes sovereign-aho-corasick. — ✅ **integrated**: used by `sovereign-token-law-fuse`
- **`sovereign-token-law-fuse`** — The checkpoint-free token-law fusion primitive (M00155 / SDD-507): compose the named constraint planes — grammar (JSON-schema) / positive-regex / literal-denylist / negated-regex / static-policy — at a given generated prefix into ONE vocab allow-mask, WITHOUT a transformer. The fused mask is the deterministic-cortex DECISION (which next tokens every active law permits); it depends only on the layer sources + the vocabulary strings, never on trained weights, so it is exact regardless of which checkpoint (or none) is loaded. sovereign-llm's complete_with_token_law consumes this per decode step; a data-plane HTTP route and the osctl surface (F00792/F00795/F00797) drive it directly. Composes sovereign-token-law-mask + sovereign-json-schema-grammar + sovereign-token-grammar-mask + sovereign-regex-constrain + sovereign-token-law-deny. — ✅ **integrated**: used by `sovereign-gatewayd`
- **`sovereign-token-law-mask`** — The M002 token-law bitset as a decode-time logit mask (SDD-500). ## 4. Demo-hub-only libraries (22) — reached only via the island hubs

Consumed only through `sovereign-llm` / `sovereign-retrieval`, which the daemon does not use — so nothing that runs reaches them. Wiring a hub into production, or giving these real consumers, is the open work (audit F-2026-083/088/089).
- **`sovereign-answer-extract`** — Final-answer extraction from chain-of-thought output: pulls the answer out of a model's reasoning by honoring 'Answer:'/'Final answer:'/'the answer is' markers (falling back to the last line), and extracts the last numeric value for math tasks. The step between reasoning and a usable answer — pairs with self-consistency voting.
- **`sovereign-best-of-n`** — Inference-time scaling by sampling N candidates and picking a winner: best-of-N by a score (logprob or reward), top-k selection, and weighted self-consistency that sums each distinct answer's candidate scores and takes the highest. The aggregation layer for spending more compute at decode time to get a better answer.
- **`sovereign-confidence-calibration`** — Confidence calibration for model outputs: temperature scaling (fit a single temperature that minimizes negative log-likelihood) plus calibration diagnostics — Expected Calibration Error and Brier score. So a runtime can trust, threshold, or abstain on predicted probabilities instead of taking raw softmax confidence at face value.
- **`sovereign-context-budget`** — Context-window budgeting: counts a text's tokens with the runtime tokenizer and trims it to fit a token budget, keeping the head or the tail. The bookkeeping that stops an agent's growing prompt from overflowing the model's context. Composes sovereign-tokenizer.
- **`sovereign-decoder-stack`** — A complete decoder-only transformer model: token embedding -> N stacked pre-norm decoder blocks (each with its own KV cache) -> final RMSNorm -> output head -> logits -> sampler, with prompt ingestion and autoregressive generation. The end-to-end tiny LLM runtime that ties the whole decode-engine arc together. Composes sovereign-transformer-block, sovereign-rmsnorm, and sovereign-sampler.
- **`sovereign-degeneration`** — Generation-degeneration detection: flags the loop/repeat collapse that decoding can fall into, by measuring the longest repeated substring (via a suffix array), the distinct-n-gram ratio (rep-n), and the fraction of text covered by repeats. A runtime quality gate for catching degenerate output. Composes sovereign-suffix-array.
- **`sovereign-diversity`** — Measure the diversity of a set of generations: distinct-n (fraction of unique n-grams across outputs), Self-BLEU (how much each output overlaps the others — lower is more diverse), and the unique-output ratio. For detecting mode collapse and tuning sampling temperature. Composes sovereign-text-eval.
- **`sovereign-dry-sampler`** — DRY (Don't Repeat Yourself) repetition sampler: penalizes the next token in proportion to the length of the repeated sequence it would extend, with an exponentially growing penalty above an allowed match length, and configurable sequence-breaker tokens that reset matching. Unlike a flat repetition penalty or a hard n-gram ban, it targets exactly the token that continues a loop — suppressing degenerate repetition while leaving legitimate reuse untouched.
- **`sovereign-gumbel`** — Gumbel-max sampling from raw logits: adding i.i.d. Gumbel noise and taking the argmax draws a category exactly from the softmax of the logits, with no explicit normalization — and taking the top-k instead draws k distinct categories without replacement (the Gumbel-top-k trick). Seeded and deterministic, for categorical sampling and diverse parallel draws.
- **`sovereign-json-extract`** — Robust JSON extraction from model output: scans text for the first balanced JSON object or array — correctly ignoring braces inside strings and escapes — and returns its span or parses it to a serde_json Value. The boundary that turns a model's prose-wrapped JSON (tool arguments, structured answers) into usable data.
- **`sovereign-llm`** — The end-to-end sovereign LLM runtime: ties the byte-level BPE tokenizer to the decoder-only transformer stack so you can go straight from a text prompt to generated text. tokenizer.encode -> decoder-stack.generate -> tokenizer.decode, deterministic per seed. The runnable text-to-text capstone of the inference engine. Composes sovereign-tokenizer and sovereign-decoder-stack.
- **`sovereign-logprobs`** — Utilities over token log-probabilities: sequence log-prob, length-normalized scoring (GNMT penalty), perplexity, predictive entropy, top-token confidence margin, and the weakest-token locator for hallucination flagging. The scoring-and-uncertainty layer that reads a model's per-token logprobs.
- **`sovereign-perplexity`** — Language-model evaluation by perplexity: scores a model's per-token log-likelihood on a reference token sequence and reports cross-entropy and perplexity. The standard intrinsic LM metric, used to compare models or quantization levels. Composes sovereign-decoder-stack (its Clone-able model is scored without mutating the original).
- **`sovereign-prompt-compress`** — Selective context compression: drop the most predictable (low-surprisal) tokens and keep the informative ones to fit a token budget, preserving order — the LLMLingua approach to cutting long-context cost. Works from per-token log-probabilities or an entropy/budget target.
- **`sovereign-repetition-penalty`** — Soft anti-repetition sampling penalties applied to logits from the generated token history: CTRL-style repetition penalty (scale), OpenAI-style frequency penalty (subtract per occurrence) and presence penalty (subtract once if seen). Discourages loops and verbatim repeats without hard-banning tokens — distinct from no-repeat-ngram and logit masking.
- **`sovereign-self-consistency`** — Self-consistency decoding: draw several answers (varying the seed), normalize them, and return the majority vote with its agreement fraction — trading extra compute for reliability on reasoning tasks. Generic over any answer generator, so it wraps the real runtime or a scripted one.
- **`sovereign-semantic-cache`** — A semantic completion cache: stores completions keyed by their prompt's embedding and returns a cached one for any new prompt whose embedding is within a cosine-similarity threshold — so near-duplicate requests (paraphrases, typos) hit the cache, not just exact repeats. The fuzzy counterpart to the exact-match completion cache. Composes sovereign-embed.
- **`sovereign-suffix-array`** — Suffix array over a byte string with prefix-doubling construction and Kasai LCP: index every suffix once, then count or locate any substring by binary search in O(m log n), and find the longest repeated substring from the LCP array. For detecting degenerate verbatim repetition in generations and fast substring queries over a fixed text.
- **`sovereign-text-eval`** — Generation-quality metrics: BLEU (clipped n-gram precision with brevity penalty) and ROUGE-N / ROUGE-L (n-gram and longest-common-subsequence overlap) for scoring generated text against one or more references. The standard kit for evaluating summaries, translations, and completions.
- **`sovereign-token-healing`** — Token healing for generation: when a prompt ends mid-token, trim the trailing token(s) and constrain the first generated token to be consistent with the removed text, so the model re-chooses a natural boundary instead of being locked into a bad split. Fixes the prompt/completion seam for completion and constrained decoding.
- **`sovereign-typical-sampling`** — Locally typical sampling (Meister et al.): keep the smallest set of tokens whose surprisal is closest to the distribution's conditional entropy and whose probability sums to a target mass, then sample from those. A truncation method distinct from top-k / top-p / min-p, aimed at more human-like, less degenerate generation.
- **`sovereign-xtc-sampler`** — XTC (Exclude Top Choices) sampler: with a set probability, removes the most-probable tokens whenever several already clear a probability threshold, keeping only the least-probable confident option plus the long tail. Boosts diversity and creativity without the incoherence of high temperature — it only fires when there is a safe alternative, so a single forced token is never excluded. Operates as a logit mask with a seeded activation roll.

- **`sovereign-sprt`** — Wald's sequential probability ratio test for Bernoulli outcomes: accumulate the log-likelihood ratio one observation at a time and stop the moment the evidence crosses an accept-H0 or accept-H1 boundary set by the target error rates. Decides whether a success rate is closer to p0 or p1 with far fewer samples than a fixed-size test — for early-stopping online evaluation and A/B comparisons.
- **`sovereign-sse-parse`** — Streaming Server-Sent Events parser: feed it response chunks as they arrive and it yields complete events (event/data/id/retry fields, multi-line data joined with newlines), buffering partial events across chunk boundaries. For consuming the SSE wire format that streaming LLM APIs use.
- **`sovereign-suffix-automaton`** — Suffix automaton: the minimal deterministic automaton recognizing every substring of a string, built online in linear time. Answers substring membership in O(pattern), counts the distinct substrings of the text in O(n), and finds the longest common substring with another string in one pass. For substring search, near-duplicate and overlap analysis, and string indexing — distinct from a suffix array.
- **`sovereign-task-lifecycle`** — E0548 / E0556 — the Task Lifecycle: the 12 lifecycle steps (Intake … Resume/Archive) and the 9 task states (active/paused/waiting_user/waiting_tool/hibernated/completed/failed/rolled_back/archived) with a validated state machine + resume requirements.
- **`sovereign-task-map`** — E0551 — Map (lifecycle step 4): the 4 domain-specific maps (code / research / GUI / OS-admin) and the components each gathers. 'MAP prevents blind action.'
- **`sovereign-tdigest`** — t-digest streaming quantile estimator: clusters values into centroids whose size is bounded by a scale function, so resolution is finest at the extremes (p1, p999) where it matters and coarse in the middle. Gives rank-error-bounded quantiles over an unbounded value range with a small fixed footprint, and merges across shards — the Dunning t-digest used in Elasticsearch and friends. Complements relative-error DDSketch with a different accuracy profile.
- **`sovereign-telemetry-backend`** — Operator-overrideable telemetry sink selection matching the applied profile schema observability.telemetry_sink (prometheus-local / otel / none, SDD-004 Q-013 + SDD-016): token parsing for the SOVEREIGN_TELEMETRY_SINK env var and the --telemetry-sink CLI flag, plus the CLI > env > profile > default resolution precedence.
- **`sovereign-text-edit`** — Precise find-and-replace text editing: apply edits that each require their target to occur exactly once (rejecting ambiguous or missing matches), the safe edit model a code-editing agent uses to change a file without clobbering the wrong spot. The apply side of the line-diff.
- **`sovereign-token-meter`** — Token accounting and cost metering: tracks input/output tokens against optional budgets, enforces limits before a spend, and estimates cost from per-1k prices. The bookkeeping behind the sovereign runtime's $0-target, cost-aware routing — what tells you whether a request fits the budget before you run it.
- **`sovereign-tool-catalog`** — Runtime tool catalog — curated registry of cockpit-callable tools (shell / fs-read / fs-write / web-fetch / model-inference / mcp-bridge / replay-control / cli-bridge). Each entry declares side-effect class + minimum execution mode + required profile bundle so the cockpit can gate availability per session context.
- **`sovereign-tool-invocation-record`** — Runtime per-invocation record for sovereign-tool-catalog tools — each cockpit-issued tool call emits one immutable record (tool, mode, bundle, started_at, completed_at, exit_kind, bytes_out, trace_id). The replay engine reconstructs the cockpit session from this stream.
- **`sovereign-trust-boundaries`** — M014 — isolation and trust boundaries: the 4 trust zones (Host Control / Oracle / Scout-Sandbox / Disposable) and the 4-tier tool sandbox ladder A/B/C/D, with the tier→minimum-zone containment rule that keeps untrusted tools off the host.
- **`sovereign-typed-state`** — E0557 — the Critical Data Flow Law: 'Text is not the system state. Text is payload inside typed state.' The real task state as 8 typed components (frames/routes/policies/memory_refs/tool_observations/eval_results/commits/traces). Integrates sovereign-trace-context for traces + commits.
- **`sovereign-ulid`** — ULID generation: 128-bit identifiers as a 48-bit millisecond timestamp plus 80 bits of randomness, encoded in 26 Crockford base32 characters that sort lexicographically by creation time. Monotonic within a millisecond. For request and trace ids that are unique, time-ordered, and URL-safe — a sortable alternative to UUIDs.
- **`sovereign-unigram-tokenizer`** — Unigram-LM subword tokenization (the SentencePiece scheme): given a vocabulary of pieces with log-probabilities, segment text into the maximum-likelihood sequence of pieces with a Viterbi-style dynamic program over the piece lattice. A third tokenizer distinct from byte-pair encoding and WordPiece, with single-character fallback so every input is segmentable.
- **`sovereign-varint`** — LEB128 variable-length integer coding with zigzag for signed values: encode small integers in one byte and large ones in as many as needed, plus delta-encoding helpers for monotonic sequences. For compactly serializing token-id streams, offsets, and posting lists without fixed-width waste.
- **`sovereign-vector-clock`** — Causality tracking for distributed events: a vector clock that decides whether one event happened-before another or whether the two are concurrent (the question a scalar timestamp cannot answer), plus a Lamport clock for a cheap total order consistent with causality. For ordering events across replicas, detecting conflicts, and driving CRDT or gossip reconciliation.
- **`sovereign-viterbi`** — Viterbi decoding for hidden Markov models: the log-space dynamic program that finds the single most likely hidden-state sequence behind a series of observations in O(T·S^2), with backpointer reconstruction. For sequence labeling, segmentation, and Unigram-LM-style most-probable tokenization.
- **`sovereign-watermark`** — LLM text watermarking (green-list scheme): at each step a context-seeded hash splits the vocabulary into a 'green' set that gets a logit boost, so generated text carries a statistical signature; detection counts green tokens and reports a z-score against the null hypothesis. For provenance — telling whether text came from a watermarked model — without storing the text.
- **`sovereign-weighted-reservoir`** — Weighted reservoir sampling (Efraimidis-Spirakis A-Res): keep a size-k sample from an unbounded stream where each item is kept with probability proportional to its weight, in O(k) space and a single pass. For importance-weighted sampling of logs, examples, or generations. Seeded and deterministic.
- **`sovereign-wer`** — Word Error Rate and Character Error Rate: the transcription/ASR metric (substitutions + insertions + deletions over reference length) with the full S/I/D breakdown, built on edit-script alignment. For scoring transcription, normalization, and any reference-vs-hypothesis text comparison. Composes sovereign-edit-align.
- **`sovereign-wordpiece`** — WordPiece tokenization (the BERT scheme): greedy longest-match subword segmentation against a fixed vocabulary, emitting '##'-prefixed continuation pieces and falling back to an [UNK] token for words it cannot cover. A real tokenizer variant distinct from byte-pair encoding, with detokenization back to text.
- **`sovereign-worker-status-word`** — E0111 / M00212 — the worker 64-bit status word. Eight byte-fields (load / memory / thermal / queue / error / health / policy_mode / flags) packed into one u64 the engine writes and the scheduler reads. 'Telemetry becomes bits — each worker gets a status word.'
- **`sovereign-workflow-graph`** — E0552 — Plan/Compile (lifecycle step 5): the workflow graph. The 8 node types (model-call / tool-call / memory-read / test-run / policy-gate / human-gate / eval / commit) and a DAG of dependency edges, with cycle detection + topological execution order.
- **`sovereign-workspace-folder-registry`** — Runtime workspace-folder registry — operator-declared root folders the cockpit + tool catalog scope filesystem operations to. Each folder declares (label, root_path, scope, read_only, max_size_gb). The dispatcher refuses fs-read/write outside any declared folder.
- **`sovereign-zfs-dataset-layout`** — M068 — canonical ZFS dataset layout for the tank pool (context / containers / models / logs / snapshots / vault) with per-dataset recordsize, compression and sync properties, plus a drift validator that flags an observed `zfs get` layout against the catalogued canon (E0660-E0664, F05702-F05730).
